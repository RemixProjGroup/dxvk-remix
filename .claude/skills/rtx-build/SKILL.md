# rtx-build

Wraps the proper build incantation for this dxvk-remix port so Claude
doesn't have to re-derive the cleanup-steps + PerformBuild flags each
time.

## When to invoke

- User asks to build, compile, compile-check, verify, or smoke-test
- Before pushing code changes to `modern-games-sdk-api` per
  end-of-project discipline
- After a code change that touches upstream files or `rtx_fork_*`
  modules

## When NOT to invoke

- Doc-only changes (`.md`, `CLAUDE.md`, `docs/`, `.claude/`
  entries) — no compile impact
- Skill / settings / config edits
- Before the user has finished their code edits — building against a
  work-in-progress tree wastes time

## Steps

1. Run this PowerShell command with `run_in_background: true`
   (expected duration: 1-3 min incremental, 10-15 min cold full):

   ```powershell
   Remove-Item -Path .\nv-private -Recurse -Force -Confirm:$false -ErrorAction SilentlyContinue
   Remove-Item -Path .\tests\rtx\dxvk_rt_testing -Recurse -Force -Confirm:$false -ErrorAction SilentlyContinue
   . .\build_common.ps1
   PerformBuild -BuildFlavour release -BuildSubDir _Comp64Release -Backend ninja -EnableTracy false
   ```

   (Run from the project root — the working directory Claude Code
   starts in is already correct.)

2. Wait for the task-completion notification — DO NOT poll, the
   runtime will wake you up when the background command exits.

3. When the task completes, read the output log file (path is in the
   task notification) and scan for errors:

   ```bash
   grep -iE "error C[0-9]|fatal error|FAILED:|Failed to run" <output-log>
   ```

4. Report back:
   - **Green (exit 0 + no error lines):** "Build exit 0, zero compile
     errors." Tail last ~5 lines of the log to confirm install phase
     completed.
   - **Red:** show first 10 error lines + the files they came from.
     Let the user see them before attempting fixes.

## Alternate flavors

If the user explicitly asks for a different build flavor, substitute
the corresponding `BuildSubDir`:

| Flavour | BuildSubDir |
|---|---|
| release (default) | `_Comp64Release` |
| debug | `_Comp64Debug` |
| debugoptimized | `_Comp64DebugOptimized` |

## Gotchas

- **`nv-private/` and `tests/rtx/dxvk_rt_testing/`** are gitignored
  and regenerated by meson's install step. They MUST be removed
  before each build, or install can get confused on the next run.
  The `-ErrorAction SilentlyContinue` on the Remove-Item calls makes
  it safe to run when they don't exist yet.
- **`LNK4098: defaultlib 'LIBCMT' conflicts`** is a pre-existing
  upstream warning — not a failure.
- **`WARNING: msvc does not support C++11; attempting best effort`**
  is also upstream, not a failure.
- **Must run under PowerShell (not bash).** `build_common.ps1` dot-
  sources `vcvars64.bat` and sets up the VS2019 environment via
  `vswhere.exe`.
- The build produces `d3d9.dll` in `_Comp64Release/src/d3d9/` and
  also installs it to `_output/`, `public/bin/`, and the test dirs.
